# Query Design

When building search queries:

1. Start with the task phrase and one or two canonical keywords.
2. Add benchmark, domain, or modality terms only if recall is too broad.
3. Search both exact phrases and looser variants.
4. If the field is fast-moving, bias toward the last 2-4 years.
5. Keep a small slot for foundational papers.

Example query shapes:
- "repo-level coding agent benchmark"
- "repository-level code generation benchmark"
- "software engineering agent evaluation benchmark"
- "arXiv repo-level coding agent"
